# Input Data Requirements

CanRad requires various spatial datasets depending on which model (L2R, C2R, or T2R) you are using. This page describes all input files, their formats, and when they are required.\
<br>
**All input files must be on the same coordinate reference system (CRS).**

## Overview of Required Inputs

| Input Type | L2R | C2R | T2R | Description |
|------------|-----|-----|-----|-------------|
| **Terrain** | Optional | Optional | Required | Digital terrain models for topographic shading |
| **Canopy** | Required | Required | — | Forest structure data (lidar or CHM) |
| **Meteorological** | Optional | Optional | Optional | Above-canopy radiation measurements |

## Terrain Input Files

Terrain data is used to calculate topographic shading effects on radiation transmission. Two scales of terrain can be incorporated:

### High-Resolution Terrain

> **Variable:** `hrdtmf`  
**Models:** L2R, C2R, T2R  
**Format:** GeoTIFF  
**Description:** High-resolution digital terrain model (DTM) for calculating the near-field horizon line  
**Typical resolution:** 1-25 m  
**Spatial extent:** Covers area within `highres_peri` (default 300 m) of all calculation points  
**When to use:** Enable with `terrain_highres = true` to include local topographic effects

### Low-Resolution Terrain

> **Variable:** `lrdtmf`  
**Models:** L2R, C2R, T2R  
**Format:** GeoTIFF  
**Description:** Low-resolution DTM for calculating the distant topographic horizon line  
**Typical resolution:** >25 m (e.g., 25-100 m)  
**Spatial extent:** Covers area within `lowres_peri` (default 25,000 m = 25 km) of calculation points  
**When to use:** Enable with `terrain_lowres = true` to include regional mountain shading effects

### Terrain Mask

> **Variable:** `termf`  
**Models:** L2R, C2R, T2R  
**Format:** NetCDF  
**Description:** Pre-computed terrain mask defining areas where terrain calculations are needed  
**How to create:** Automatically generated by CanRad when `terrain_precalc = true`, or output from T2R  
**When to use:** Speeds up repeated model runs by reusing terrain calculations

### Horizon Line Matrix

> **Variable:** `hlmf`  
**Models:** L2R, C2R, T2R  
**Format:** NetCDF  
**Description:** Pre-calculated horizon line matrix containing zenith angles of topography above the horizon for all azimuth directions  
**How to create:** Calculated by CanRad when `save_horizon = true`, or externally  
**When to use:** Set `hlm_precalc = true` to use pre-computed horizon lines (useful for consistency with topographically-downscaled radiation data)

## Canopy Input Files

Canopy data provides the 3D forest structure information needed to calculate radiation transmission through vegetation.

### Canopy Height Model (C2R only)

> **Variable:** `chmf`  
**Model:** C2R  
**Format:** GeoTIFF  
**Description:** Raster of canopy heights (height above ground)  
**Typical resolution:** 1-25 m  
**Data source:** Derived from lidar, photogrammetry, or satellite data  
**Preprocessing:** Should represent maximum canopy height within each pixel

### Lidar Point Cloud (L2R only)

> **Variable:** `lasf`  
**Model:** L2R  
**Format:** LAS or LAZ (compressed)  
**Description:** 3D point cloud from airborne lidar  
**Classification:** Ideally classified with ground (class 2) and vegetation (classes 3-5) points  
**Point density:** Higher density improves accuracy (>10 points/m² recommended)  
**Coverage:** Must extend `forest_peri` (default 100 m) beyond all calculation points

### Leaf Area Volume Density (C2R only)

> **Variable:** `lavdf`  
**Model:** C2R  
**Format:** GeoTIFF  
**Description:** 3D distribution of leaf area within the canopy  
**How to create:** Output from `canrad_precalc.jl` preprocessing script  
**Units:** Leaf area per unit volume (m²/m³)  
**Required:** Essential for C2R to represent vertical canopy structure

### Tree Information File

> **Variable:** `dbhf`  
**Models:** L2R, C2R  
**Format:** Plain text (`*_treeinfo.txt`)  
**Description:** Individual tree attributes including diameter at breast height (DBH)  
**How to create:** Output from `canrad_precalc.jl` preprocessing with pycrown tree detection  
**When to use:** Required when `trunks = true` to add opaque trunks to synthetic images  
**Content:** Tree ID, location (x, y), height, crown radius, DBH

**Example format:**
```
TreeID  X           Y           Height  CrownRadius  DBH
1       2647123.45  1185678.90  15.2    3.1          0.25
2       2647128.12  1185682.34  18.6    3.8          0.32
```

### Lidar Tree Classification (L2R only)

> **Variable:** `ltcf`  
**Model:** L2R  
**Format:** LAZ (compressed LAS)  
**Description:** Lidar point cloud with additional classification identifying which points belong to specific tree crowns  
**How to create:** Output from pycrown tree detection, then processed by `canrad_precalc.jl`  
**When to use:** Required when `trunks = true` for L2R  
**Note:** Links lidar points to individual trees in the tree information file

### Canopy Base Height (C2R only)

> **Variable:** `cbhf`  
**Model:** C2R  
**Format:** GeoTIFF (same resolution as `chmf`)  
**Description:** Height of the canopy base (lowest branch height) above ground  
**How to create:** Output from `canrad_precalc.jl` or estimated as fraction of tree height  
**When to use:** Enable when crown base height varies spatially (improves vertical canopy structure)  
**Alternative:** Can use fixed ratio (e.g., 60% of tree height) via settings instead

## Meteorological Input Files

### Shortwave Radiation Data

> **Variable:** `swrf`  
**Models:** L2R, C2R, T2R  
**Format:** Plain text, tab-delimited  
**Description:** Above-canopy shortwave radiation measurements  
**When to use:** Required only when `calc_swr = 2` (calculate actual radiation rather than potential)  
**Columns:** 
> 1. DateTime (format: YYYY-MM-DD-HH:MM:SS or as specified)
> 2. Shortwave radiation value (W/m²)

**Example format:**
```
2023-06-01-00:00	0.0
2023-06-01-00:30	0.0
2023-06-01-01:00	2.3
```

**Important requirements:**
- Time steps must exactly match the `tstep` setting in the configuration
- Temporal coverage must span from `startdate` to `enddate` specified in settings
- Missing values are not allowed; use gap-filling if needed

## File Naming and Organization

### Recommended Directory Structure

```
project/
├── datasets/
│   ├── DTM_highres.tif
│   ├── DTM_lowres.tif
│   ├── CHM.tif
│   ├── LAVD.tif
│   ├── CBH.tif
│   ├── treeinfo.txt
│   └── meteo/
│       └── SWR_measured.txt
├── settings/
│   └── C2R_settings.jl
└── output/
```

### File Path Specification

All input file paths are specified in the settings file (e.g., `C2R_settings.jl`). You can use absolute paths or specify paths relative to the working directory which can be set as an input when calling the settings:

```julia
function chm2rad_settings(wrkdir::String)
    
    ...

    dat_in = Dict(
        # chm
        "chmf" => wrkdir*"/datasets/test_chm.tif",
        # dtm -> high res terrain
        "hrdtmf" => wrkdir*"/datasets/test_dtm.tif"
    )

    return dat_in, par_in
```

## Data Preparation Workflow

### For C2R

1. **Prepare base canopy data:**
   - Obtain canopy height model (CHM) or calculate from airborne lidar data

3. **[optional] Run `canrad_precalc.jl`:**\
    - Runs pycrown for tree detection: requires CHM, DTM, DSM (digital surface model), and [optional] point cloud
   - Processes pycrown outputs (tree crown polygons and attributes)
   - Generates: LAVD file, tree info file, canopy base height

### For L2R

1. **Classify lidar point cloud:**
   - Ensure ground points (class 2) and vegetation points (classes 3-5) are identified
   
2. **Optional preprocessing with pycrown:**
   - Required only if including trunks (`trunks = true`) or adding branches (`branches = true`)
   - Follow documentation in canrad_precalc.jl for steps to generate tree info and classified point cloud
   
### For T2R/C2R/L2R

1. **Prepare terrain data** at appropriate resolution(s)

### For All Models

1. **Prepare model run points:**
   - Create a text file with x, y coordinates of calculation points (e.g., `pts.txt`)
   - Ensure points fall within the spatial extent of all input data

2. **Configure settings file** with path to points file and all required inputs

3. **Prepare run file** (e.g., `run_canrad.jl`) to load settings and execute the model

## Coordinate Reference Systems

**Important:** All spatial inputs must share the same coordinate reference system (CRS):
- Terrain rasters (GeoTIFF files)
- Canopy rasters (GeoTIFF files)  
- Lidar point clouds (LAS/LAZ files)
- Calculation point locations

CanRad does not perform CRS transformations. Reproject all data to a common projected coordinate system (e.g., UTM) before running the model.

## Data Quality Recommendations

| Data Type | Recommendation | Impact |
|-----------|---------------|---------|
| **CHM resolution** | 1-5 m for C2R | Higher resolution improves spatial detail but increases computation |
| **Lidar point density** | >10 pts/m² for L2R | Higher density improves canopy structure representation. Very low densities (1-10 pts/m² are probably too low for lidar point cloud enhancement) |
| **High resolution terrain** | Match application scale | 1-5 m for points and high (1-2m) point-spacing at gridded scale; 10-25 m for larger-scale studies (if using) |
| **Low resolution terrain** | Match application scale | 50-100 m |
| **Tree detection** | Validate pycrown parameters | Poor segmentation (particularly underestimation) will impact where lidar point density can be increased  |
| **SWR measurements** | Quality-controlled, gap-filled | Errors propagate directly to radiation outputs |

## Troubleshooting Common Issues

**Error: "Coordinate system mismatch"**
- Ensure all spatial inputs share the same CRS
- Check that calculation points fall within input data extents

**Poor canopy representation:**
- For L2R: Check tree crown segmentation and trunk positions
